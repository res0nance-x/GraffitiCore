Help & Documentation

What is Graffiti?

Graffiti is a secure, decentralized, peer-to-peer (P2P) messaging application. It allows you to communicate directly with other nodes without central servers. All messages are encrypted end-to-end, ensuring that only the intended recipient can read the content.

Server & Connections

To receive direct connections, you can start a Server on your node. This opens a port (typically 1337) so that other peers on the network can connect to you, handshake, and synchronize messages.

Peer Discovery & Syncing

Graffiti discovers local network nodes automatically using multicast discovery. You can also manually connect to any reachable peer by inputting their IP/host and port in the Network tab. Once connected, Graffiti automatically runs an initial sync, exchanging encrypted message metadata so both nodes get caught up.

Topic / Relay Workflow

To start messaging in a shared topic/forum using a relay node, follow this standard flow:

  1. Setup the Shared Topic: Create or import the target Identity using its deterministic seed phrase. Ensure this identity is active in your "Identities" tab.
  2. Import Peers: Add the public keys of the peers you want to send messages to in the "Peers" tab.
  3. Connect to the Relay: In the "Network" tab, input the host/IP and port of the relay node and click Connect. Alternatively, scan and connect via "Nearby Nodes".
  4. Write and Send: Go to the "Messages" tab. Compose your message, selecting your sender identity in the "From" field and the shared/recipient peer in the "To" field. Click Send.
  5. Propagation: The message is encrypted locally, stored on your disk, and automatically pushed to the connected relay. The relay caches the encrypted payload and serves it to the recipient when they connect and synchronize.

Relay Mode

If you enable Relay mode, your node acts as a store-and-forward peer. It will download and cache encrypted messages destined for other identities and pass them along when those nodes connect to you. This is highly useful for bridging communication when peers are not online at the same time.

Identities & Passphrases

In Graffiti, accounts are called Identities.

  • Receiving Messages: You can only read and decrypt messages addressed to an Identity key that you currently have active in your "Identities" tab.
  • Seed Phrases: Identities are created deterministically using a seed phrase (password/passphrase). If you share this seed phrase with someone else (or use it on a second device), that device will recreate the exact same Identity and can view and receive the same message history.
  • Forums & Uniqueness: You can uniquely identify yourself to others by creating your own private Identity and using it to write to forum topics. Emojis and avatar colors represent the unique fingerprint of each identity key.
